== 2.1 Technical Constraints

[cols="1,3"]
|===
|Constraint |Explanation

|Rust Programming Language
|Chosen for performance, reliability, and ecosystem. Minimum supported version is 1.70.

|JSON Storage Format
|Required for human readability and easy integration with other tools. Files stored in `.clicky/` directory.

|CLI-First Design
|All functionality must be accessible via command-line arguments before TUI features are added.

|Cross-Platform Support
|Must work on macOS, Linux, and Windows without platform-specific code in core logic.
|===

== 2.2 Organizational Constraints

[cols="1,3"]
|===
|Constraint |Explanation

|Open Source
|MIT OR Apache-2.0 dual license for maximum compatibility.

|Clean Architecture
|Must follow Clean Architecture principles with clear separation of concerns.

|Test-Driven Development
|New features require tests; bug fixes require regression tests.

|Documentation
|arc42 documentation must be maintained alongside code.
|===

== 2.3 Conventions

[cols="1,3"]
|===
|Convention |Explanation

|Card ID Format
|Three-letter prefix + sequential number (e.g., PRJ-001). Prefix derived from board name.

|Default Columns
|New boards created with: "To Do", "In Progress", "Done"

|Directory Structure
|One board per directory, stored in `.clicky/board.json`

|Exit Codes
|0 = Success, 1 = General Error, 2 = Board Not Found, 3 = Invalid Input

|Date Format
|ISO 8601 format in storage, human-readable format in output
|===
